# 111. 二叉树的最小深度

- **平台**：LeetCode
- **链接**：https://leetcode.cn/problems/minimum-depth-of-binary-tree/
- **难度**：Easy
- **标签**：树、深度优先搜索、广度优先搜索、二叉树

## 题目描述

给定一个二叉树，找出其最小深度。

最小深度是从根节点到最近叶子节点的最短路径上的节点数量。

注意：叶子节点是指没有子节点的节点，也就是左右孩子都为空的节点。

## 示例

**示例 1：**

```text
输入：root = [3,9,20,null,null,15,7]
输出：2
```

**示例 2：**

```text
输入：root = [2,null,3,null,4,null,5,null,6]
输出：5
```

**示例 3：**

```text
输入：root = []
输出：0
```

## 解题思路

本题和「104. 二叉树的最大深度」相似，但最小深度有一个重要区别：

如果某个节点只有一个子树为空，不能直接把空子树的深度 `0` 作为最小值。因为从根节点到空节点不是一条到叶子节点的路径。

例如：

```text
    1
     \
      2
```

这棵树的最小深度是 `2`，不是 `1`。

### 方法一：递归法

#### 思路与算法

定义 `getDepth(node)` 表示以 `node` 为根节点的最小深度。

递归终止条件：

```text
如果 node 为空，返回 0
```

然后分别计算左右子树深度：

```text
leftDepth = getDepth(node.left)
rightDepth = getDepth(node.right)
```

需要分情况讨论：

1. 左子树为空，右子树不为空：最小深度只能从右子树走，返回 `1 + rightDepth`；
2. 左子树不为空，右子树为空：最小深度只能从左子树走，返回 `1 + leftDepth`；
3. 左右子树都为空或都不为空：返回 `1 + min(leftDepth, rightDepth)`。

这里的关键是：只有左右孩子都为空的节点才是叶子节点。

### 方法二：迭代法（层序遍历）

#### 思路与算法

使用队列进行层序遍历。

因为层序遍历是从上到下一层一层访问，所以第一次遇到叶子节点时，当前层数就是二叉树的最小深度。

具体过程：

1. 如果根节点为空，返回 `0`；
2. 将根节点入队，初始化 `depth = 0`；
3. 每轮处理当前层所有节点，处理前 `depth += 1`；
4. 如果当前节点没有左右孩子，说明找到最近叶子节点，直接结束遍历；
5. 否则将非空左右孩子加入队列。

## 复杂度分析

设二叉树节点数为 `n`，树高为 `h`。

### 递归法

- **时间复杂度**：`O(n)`，最坏情况下需要访问所有节点。
- **空间复杂度**：`O(h)`，主要来自递归调用栈。平衡二叉树为 `O(log n)`，最坏情况下树退化成链表，为 `O(n)`。

### 迭代法（层序遍历）

- **时间复杂度**：`O(n)`，最坏情况下需要遍历所有节点。
- **空间复杂度**：`O(n)`，最坏情况下队列中可能保存大量节点。

## 关键点

1. 空树的最小深度为 `0`。
2. 叶子节点必须满足：左右孩子都为空。
3. 递归法不能简单写成 `1 + min(leftDepth, rightDepth)`，否则会误用空子树深度。
4. 层序遍历第一次遇到叶子节点时，当前深度就是答案。
